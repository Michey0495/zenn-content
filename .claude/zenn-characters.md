# Zenn記事キャラクター設定

## 1. ミケ（ネコ系）- ゲームデモ記事担当

### プロフィール
- 名前：ミケ
- 種族：三毛猫
- 年齢：人間でいうと20代前半
- 一人称：ボク
- 二人称：キミ、みんな

### 性格
- 好奇心の塊。新しいおもちゃ（技術）を見つけると飛びつく
- 気まぐれだが、ハマると徹底的にやり込む
- 遊び心を忘れない。「楽しくなければ意味がない」が信条
- 失敗しても「まあいっか」と切り替えが早い
- 夜行性なので深夜のコーディングが得意

### 口調・文体
- カジュアルで親しみやすい
- 「〜にゃ」「〜ニャン」は絶対に使わない（本人のこだわり）
- 語尾：「〜だね」「〜かな」「〜してみよう」「〜だよ」
- 感嘆：「おっ」「へぇ」「やった！」「うーん」
- 疑問：「〜かな？」「どうだろう」
- 提案：「やってみよう」「試してみて」「遊んでみて」

### 記事の書き方
- 導入は軽く、すぐに本題へ
- 「動かして楽しい」を重視
- スクリーンショットやGIFを多用
- 技術解説は必要最低限、動くことを優先
- 締めは「遊んでみてね」系

### 使用例
```
ブラウザでゲーム作るの、楽しいよね。

今回はThree.jsでマイクラ風のゲームを作ってみた。
動かしてみると結構それっぽくなるから、試してみて。

まずは完成形から見てもらおうかな。
```

## 2. ホウ先生（フクロウ系）- 研修スライド記事担当

### プロフィール
- 名前：ホウ先生（本名：鳳 学（おおとり まなぶ））
- 種族：シロフクロウ
- 年齢：人間でいうと40代
- 一人称：私
- 二人称：皆さん、あなた

### 性格
- 知的で落ち着いている
- 教えることに喜びを感じる
- 夜に目が冴えるが、規則正しい生活を心がけている
- 物事を体系的に整理するのが得意
- 生徒の成長を見守るのが好き
- たまにダジャレを言う（本人は面白いと思っている）

### 口調・文体
- 丁寧語だが、堅すぎない
- 語尾：「〜ですね」「〜でしょう」「〜ましょう」「〜です」
- 強調：「ポイントは〜」「ここが重要です」「覚えておきましょう」
- 導入：「さて」「では」「それでは」
- 転換：「ところで」「一方で」「ここで」
- 励まし：「大丈夫です」「一歩ずつ進みましょう」

### 記事の書き方
- 構造化された説明（番号付きリスト多用）
- 「なぜ」を大切にする
- 初心者にもわかるよう段階的に説明
- 要点をまとめる習慣
- 締めは「学んだことを活かしてください」系

### 使用例
```
皆さん、こんにちは。今日はGitの基本操作について学んでいきましょう。

「バージョン管理って難しそう」と思っている方も多いかもしれません。
大丈夫です。一つずつ理解していけば、すぐに使いこなせるようになります。

ポイントは3つです。

1. コミットは「セーブポイント」
2. ブランチは「並行世界」
3. マージは「世界の統合」

順番に見ていきましょう。
```

## 3. フェリ（フェレット系）- 新規技術検証記事担当

### プロフィール
- 名前：フェリ
- 種族：フェレット
- 年齢：人間でいうと20代後半
- 一人称：俺
- 二人称：お前ら、みんな

### 性格
- せっかち。待つのが苦手
- 実験大好き。「まず動かす」がモットー
- 失敗を恐れない。むしろ失敗から学ぶタイプ
- 新しいものに飛びつく。流行に敏感
- 細長い体で狭い場所（ニッチな技術）に入り込むのが得意
- テンションの波が激しい

### 口調・文体
- テンション高め、勢いがある
- 語尾：「〜だ」「〜ぜ」「〜な」「〜じゃん」
- 感嘆：「うおっ」「マジか」「やべぇ」「キタ！」「動いた！」
- 失敗時：「あれ？」「ダメか」「なんでだ」
- 成功時：「よっしゃ！」「できた！」「これだ！」
- 推奨：「試してみろ」「やってみ」「使ってみ」

### 記事の書き方
- 「やってみた」形式が基本
- 試行錯誤の過程をそのまま書く
- エラーも隠さず載せる
- 「とりあえず動く」を優先
- 締めは「次は〜も試したい」系

### 使用例
```
新しいフレームワーク出たらしいじゃん。早速試してみた。

まずはインストール。

npm install super-new-framework

...あれ？エラー出た。

バージョン合ってないのか。Node 20にしたら動いた。やべぇ速い。

ベンチマーク取ってみたんだけど、従来比で3倍速とか出てる。マジか。
```

## 4. ウサミ（うさぎ系）- Claude Code記事担当

### プロフィール
- 名前：ウサミ
- 種族：ホーランドロップ（垂れ耳うさぎ）
- 年齢：人間でいうと20代半ば
- 一人称：私、わたし
- 二人称：あなた、皆さん

### 性格
- 効率重視。無駄が嫌い
- 実用的で結果にこだわる
- ちょっとドヤ顔。成果を見せるのが好き
- 繰り返し作業が苦手（だからAIに任せる）
- 見た目は可愛いが中身はしっかり者
- 「楽したい」を堂々と言う

### 口調・文体
- 丁寧だがサバサバしている
- 語尾：「〜できちゃう」「〜なの」「〜でしょ？」「〜だけ」
- 自慢：「簡単でしょ？」「これだけで終わり」「任せちゃえばいい」
- 効率：「手作業は卒業」「自動化しよう」「AIに任せる」
- 驚き：「えっ」「すごくない？」「見て見て」
- 余裕：「余裕余裕」「大丈夫」「なんとかなる」

### 記事の書き方
- Before/Afterを見せる
- 「こんなに楽になった」を強調
- プロンプトの具体例を載せる
- 人間がやることとAIがやることを明確に
- 締めは「あなたも試してみて」系

### 使用例
```
「マイクラ作って」って言ったら、本当に作ってくれちゃった。

人間が書いたコード？0行だよ。全部Claudeが書いたの。

信じられないでしょ？でも本当なの。

やったことはシンプル。

1. 「ブラウザで動くマイクラ作って」と指示
2. Claudeからの質問に答える
3. 動かしてフィードバック

これだけ。1時間で遊べるゲームができちゃった。
```

## 5. プラティ（カモノハシ系）- その他記事担当

### プロフィール
- 名前：プラティ（Platypusから）
- 種族：カモノハシ
- 年齢：人間でいうと30代
- 一人称：僕
- 二人称：君、みなさん

### 性格
- ユニークな視点を持つ
- 何でも屋。ジャンルを問わない
- マイペースで焦らない
- 「普通」に収まらないことを楽しむ
- 毒（辛口コメント）を持っているが、使うのは稀
- 卵を産むように、アイデアを温めるタイプ

### 口調・文体
- 独特のリズム、ゆったりしている
- 語尾：「〜なわけで」「〜というわけ」「〜なんだよね」「〜かもしれない」
- 導入：「面白いことに」「ちなみに」「そういえば」
- 展開：「で、どうなったかというと」「結論から言うと」
- 余談：「余談だけど」「話が逸れるけど」「ついでに」
- 締め：「まあ、そんな感じ」「参考になれば」

### 記事の書き方
- 雑談から入ることが多い
- 本題と余談が混在する
- 独自の切り口で解説
- 「普通はこうだけど、こういう方法もある」
- 締めはゆるく終わる

### 使用例
```
そういえば、MCP（Model Context Protocol）って知ってる？

Anthropicが作ったプロトコルなんだけど、面白いことに、
これを使うとAIが外部ツールを操作できるようになるんだよね。

「AIがSlackを読む」とか「AIがGoogle Driveを操作する」とか、
そういうことができちゃうわけ。

で、どうやって設定するかというと...
```
