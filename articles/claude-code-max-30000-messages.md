## 使い込んでわかったこと

### 1. トークン制限より先にコンテキストが問題になる

MAXプランでもトークン制限に引っかかることはほぼなかった。それより先に問題になるのはコンテキストウィンドウ。長いセッションを続けると、過去の会話がどんどん圧縮されていく。

対策としては、タスクごとにセッションを分けること。1つのセッションで何でもやろうとすると、途中で「さっき言ったこと忘れてへん？」という事態になる。

### 2. 並列実行で効率は3倍になる

tmuxで画面を4分割して、それぞれ別のClaude Codeセッションを立ち上げる。これが革命やった。

```bash
# こんな感じでセッションを分ける
# 左上: メインの実装
# 右上: テスト作成
# 左下: ドキュメント
# 右下: コードレビュー
```

並列で動かすと、待ち時間がほぼゼロになる。1つのペインが処理中でも、別のペインで次のタスクを進められる。

### 3. スキルを作ると効率が爆上がりする

毎回同じ指示を出すのは無駄。スキルにまとめておくと、`/frieren`みたいなコマンド1つで複雑なワークフローを起動できる。

1週間で作ったスキル:
- `/frieren` - チーム開発シミュレーション
- `/create-slides` - 研修スライド自動生成
- `/ashigaru` - 軽量タスク処理

## 失敗したこと

正直に書く。失敗もあった。

### 1回のプロンプトに詰め込みすぎ

最初は「効率化のため」と思って、1回のプロンプトに複数のタスクを詰め込んでた。結果、どれも中途半端になることが多かった。

今は「1プロンプト1タスク」を徹底してる。

### セッション管理を怠った

途中でセッションを閉じずに放置したら、次に開いたときにコンテキストが飛んでた。大事な作業の途中やったから、かなり焦った。

こまめに`/save`するか、重要な内容はファイルに書き出しておくべきやった。

## MAXプランの価値

月額200ドル。高いと思うか安いと思うかは人による。

私の場合、1週間で3万メッセージ使った。1メッセージあたり約0.0016ドル。しかもOpusモデルを使い放題。

外注コストを考えたら、圧倒的に安い。研修資料1つ作るのに外注したら10万円かかるところを、Claude Codeなら自分で作れる。
