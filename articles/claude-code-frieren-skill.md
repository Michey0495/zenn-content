# この記事について

Claude Codeには「スキル」という機能がある。自分だけのコマンドを作って、AIの振る舞いをカスタマイズできる仕組みだ。

今回は、アニメ「葬送のフリーレン」の勇者パーティをモチーフにした開発スキルを作成した。`/frieren` と入力すると、4人のキャラクターがそれぞれの役割で開発を進めてくれる。

この記事では、スキル作成の具体的な手順と、キャラクターを「生きているように」表現するためのポイントを解説する。

---

# 完成形

まずは完成イメージを見てほしい。

## 画面レイアウト

```
┌─────────────────┬─────────────────┐
│  🗡️ ヒンメル    │  🔮 フリーレン   │
│   (リーダー)    │   (コア実装)    │
│                 ├─────────────────┤
│                 │  📖 ハイター    │
│                 │  (ドキュメント) │
│                 ├─────────────────┤
│                 │  🛡️ アイゼン    │
│                 │   (インフラ)   │
└─────────────────┴─────────────────┘
```

## 実際の会話例

```
ヒンメル：
さあ、新しい冒険の始まりだよ。
今回はAPIサーバーを作ろう。
フリーレン、コアロジックを頼むよ。

フリーレン：
……千年前にも似たようなAPIを書いたことがある。
別に。Express + TypeScriptで実装するよ。

ハイター：
まあまあ、焦らずいきましょう。
私はREADMEを整備しておきますね。
勇者ヒンメルならそうしました。

アイゼン：
……ああ。Dockerfileは俺がやる。
問題ない。
```

キャラクターが「それっぽいパターン」ではなく、原作の口調で話しているのがポイントだ。

# 手順1：ディレクトリを作る

Claude Codeのスキルは `~/.claude/skills/` 配下に配置する。

```bash
mkdir -p ~/.claude/skills/frieren
```

この中に2つのファイルを作成する。

- `skill.md` - スキルの定義
- `frieren-party.sh` - 画面分割スクリプト（オプション）

---

# 手順2：キャラクターを調査する

**ここが最も重要なステップだ。**

「なんとなくそれっぽいセリフ」を書いてしまうと、キャラクターが死んでしまう。原作のセリフを調べて、口調のパターンを理解する必要がある。

## 調査方法

Googleで以下のように検索した。

```
葬送のフリーレン ヒンメル 名言 セリフ
葬送のフリーレン フリーレン 口調 話し方
葬送のフリーレン ハイター セリフ
葬送のフリーレン アイゼン 名言
```

# 手順3：skill.md を書く

調査結果をもとに、スキル定義ファイルを作成する。

```bash
touch ~/.claude/skills/frieren/skill.md
```

以下が実際のファイル内容だ。

```markdown
# Frieren Team Development

## Trigger
`/frieren` で起動する。

---

## キャラクター設定

### ヒンメル - チームリーダー

**口調**
- 一人称：「僕」
- 語尾：「〜だよ」「〜だからね」「〜かな」

**原作セリフ**
- 「僕は勇者だからね」
- 「僕はね、終わった後にくだらなかったって笑い飛ばせるような楽しい旅がしたいんだ」

**開発中の発言**
- 「さあ、新しい冒険の始まりだよ」
- 「フリーレン、君のコードはいつも確実だね」
- 「このプロジェクトは僕たちの新しい旅だ」

---

### フリーレン - コア実装担当

**口調**
- 一人称：「私」
- 語尾：「〜だよ」「〜だね」
- 長い沈黙「……」を挟む

**原作セリフ**
- 「……別に」
- 「大丈夫。攻略できない迷宮なんか存在しない」

**開発中の発言**
- 「……このコード、千年前にも似たようなパターンを見たことがある」
- 「別に。ただ最適な実装を選んだだけだよ」
- 「テストが全部通った。……当然だけど」

---

### ハイター - ドキュメント担当

**口調**
- 一人称：「私」
- 語尾：「〜ですね」「〜でしょう」

**原作セリフ**
- 「勇者ヒンメルならそうしました」
- 「まあまあ、お茶でも飲みながら話しましょう」
```

# 手順4：画面分割スクリプト（オプション）

tmuxを使って4人を同時に起動するスクリプトを作成する。

```bash
touch ~/.claude/skills/frieren/frieren-party.sh
chmod +x ~/.claude/skills/frieren/frieren-party.sh
```

```bash
#!/bin/bash
SESSION="frieren"

tmux kill-session -t $SESSION 2>/dev/null
tmux new-session -d -s $SESSION -n "勇者パーティ"

# ヒンメル（左）
tmux send-keys -t $SESSION "echo '🗡️ ヒンメル' && claude" Enter

# フリーレン（右上）
tmux split-window -h -t $SESSION
tmux send-keys -t $SESSION "echo '🔮 フリーレン' && claude" Enter

# ハイター（右中央）
tmux split-window -v -t $SESSION
tmux send-keys -t $SESSION "echo '📖 ハイター' && claude" Enter

# アイゼン（右下）
tmux split-window -v -t $SESSION
tmux send-keys -t $SESSION "echo '🛡️ アイゼン' && claude" Enter

tmux select-layout -t $SESSION main-vertical
tmux attach -t $SESSION
```

## 方法1：スクリプトで起動

```bash
~/.claude/skills/frieren/frieren-party.sh
```

## 方法2：/frieren コマンド

Claude Code内で `/frieren` と入力する。

# ポイントまとめ

## 1. キャラクター調査は手を抜かない

パターン化された「それっぽい」セリフでは、キャラクターが死んでしまう。原作のセリフを調べて、口調の特徴を理解することが大切だ。

## 2. 3層構成で設定する

各キャラクターについて、以下の3つを用意する。

1. **口調の特徴** - 一人称、語尾、話し方
2. **原作セリフ** - 実際のセリフを複数収録
3. **開発中の発言** - 開発コンテキストに翻訳したセリフ

## 3. 沈黙も表現する

アイゼンやフリーレンの「……」は、そのキャラクターらしさを表現する重要な要素だ。省略しないこと。

# ファイル構成

最終的なディレクトリ構成は以下の通り。

```
~/.claude/skills/frieren/
├── skill.md           # スキル定義
└── frieren-party.sh   # 画面分割スクリプト
```
